# UI/UX Review

## Process

1. Identify the primary users, task, device range, and success outcome from available context.
2. Inspect the actual rendered interface when browser access is available; otherwise state that review is code-based.
3. Trace the critical journey through entry, action, validation, feedback, recovery, and completion.
4. Evaluate hierarchy, spacing, typography, contrast, labels, affordances, and consistency.
5. Check keyboard navigation, focus visibility/order, semantic structure, form errors, and reduced motion.
6. Check narrow, medium, and wide layouts plus zoom and long-content behavior.

## Report

Group findings by severity:

- Critical: blocks completion or excludes users.
- Major: creates likely errors, confusion, or accessibility failure.
- Minor: polish or consistency improvement.

For each finding, provide evidence, affected user/task, and a concrete recommendation. Avoid subjective redesigns without connecting them to an outcome.
